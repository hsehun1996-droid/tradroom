# 한국 증시 퀀트 트레이딩 시스템 설계서 (Blueprint)
### 추세추종 기반 멀티팩터 의사결정 엔진 — "지금 살만한 종목" 추천 + 포트폴리오 보유/교체 판단

> **이 문서의 성격**: 코드가 아니라 *전략·데이터·아키텍처 스펙*입니다. 이 문서를 Claude Code에 단계별로 넘겨 구현하는 것을 전제로 작성했습니다.
> **전제(Assumptions)**: ① 개인 투자자용(타인 대상 자문 아님) ② 백엔드는 Python ③ 한국 증시(KOSPI/KOSDAQ) 주력, 매크로는 글로벌 참조 ④ 자동매매가 아니라 **의사결정 지원(decision-support)** 도구 — 최종 주문은 사람이.
> **주의**: 세법·거래비용·규제는 매년 바뀝니다. 본문의 수치는 작성 시점 기준이며, 시행 시점에 반드시 재확인하세요.

## 0. 먼저 — 목표 함수의 모순부터 풀고 간다

요청하신 세 가지 — **"추세추종 + 높은 승률 + 낮은 리스크"** — 는 그대로는 동시에 성립하지 않습니다. 이걸 모르고 시스템을 만들면 백테스트는 화려한데 실전에서 무너집니다.

- **추세추종의 본질은 낮은 승률입니다.** 정상적인 추세추종 전략의 승률은 보통 **35~45%**입니다. 대신 이기는 거래가 지는 거래보다 훨씬 큽니다(손익비 2~4:1). 작은 손실을 여러 번 받아내고, 가끔 터지는 큰 추세를 길게 먹는 게 엣지의 전부입니다.
- **승률을 억지로 높이려 하면** 시스템은 자연히 평균회귀(mean-reversion)·역추세로 끌려갑니다. 이쪽은 승률 60~70%가 나오지만, **꼬리 리스크(tail risk)**가 숨어 있습니다. 평소 잘 먹다가 한 번에 몇 달치를 토해냅니다. "승률 높고 리스크 적다"는 착시는 대부분 이 폭탄이 아직 안 터졌을 뿐입니다.

**따라서 목표 함수를 이렇게 재정의합니다:**

> 최대화 대상: **양(+)의 기대값(positive expectancy)** = (승률 × 평균이익) − (패율 × 평균손실)
> 제약 조건: **최대낙폭(MDD) 통제** 와 **꼬리 리스크 회피**
> "승률"은 KPI가 아니라 *모니터링 지표*. 우리가 진짜 관리하는 건 **기대값·손익비·MDD·샤프/소르티노**입니다.

이 시스템에서 "리스크가 적다"는 건 *손실 자체가 드물다*가 아니라, **개별 손실이 작고(손절), 동시에 터지지 않으며(분산·상관관리), 포트폴리오 낙폭이 정해진 한도를 못 넘는다(서킷브레이커)**는 뜻입니다. 추세추종에서 안정성은 "안 지는 것"이 아니라 "질 때 적게 지는 것"에서 나옵니다.

## 1. 시스템 철학 — "Top-down 깔때기(Funnel)"

좋은 매수 후보는 *위에서 아래로* 걸러집니다. 각 단계는 다음 단계로 넘어가기 위한 **필터(gate)** 역할을 합니다.

```
[ 1. 매크로 레짐 ]  지금 위험자산을 들고 있어도 되는 국면인가? (Risk-on / Off)
        ↓ (risk-on일 때만 통과)
[ 2. 섹터/테마 ]   돈이 어디로 흐르는가? 주도 섹터 상위 N개만 사냥터로.
        ↓ (주도 섹터 안의 종목만)
[ 3. 종목 선정 ]   추세·상대강도·펀더멘탈·수급·밸류를 합산한 종합점수 상위.
        ↓ (게이트 통과 + 점수 상위만)
[ 4. 진입 타이밍 ] 지금 가격이 "사도 되는" 자리인가? (눌림/돌파/정배열)
        ↓
[ 5. 사이징·리스크 ] 얼마나 살까? 손절은 어디? 변동성 기반 비중.
        ↓
[ 6. 청산 규칙 ]    언제 팔까? 추세훼손/손절/익절/상대강도 악화.
        ↓
[ 7. 일일 모니터링 ] 보유 종목 매일 재평가 → 보유 / 축소 / 매도 / 교체.
```

핵심 원칙 3가지:
1. **위가 아래를 지배한다.** 매크로가 risk-off면 아무리 좋은 종목도 안 산다(현금/방어).
2. **추세가 없으면 사지 않는다.** 우리는 바닥을 맞히는 게 아니라 *이미 가고 있는 것*에 올라탄다.
3. **펀더멘탈은 "방향타"가 아니라 "안전벨트".** 추세가 산 종목 중에서 *지뢰(부실·실적쇼크)*를 걸러내는 용도. 깊은 가치투자(저PER 줍기)가 아니라, **추세+퀄리티의 결합(이른바 trend + quality)**.

## 2. 전체 아키텍처

레이어를 분리해야 유지보수·백테스트가 됩니다.

```
┌─────────────────────────────────────────────────────────┐
│ A. 데이터 수집 (Ingestion)   — 스케줄러(cron/airflow)      │
│    시세·재무·수급·공시·매크로·컨센서스 → 원천 저장          │
├─────────────────────────────────────────────────────────┤
│ B. 저장소 (Storage)                                       │
│    - 시계열: 가격/거래량/지표 (Parquet 또는 TimescaleDB)   │
│    - 정형: 재무/메타/포트폴리오 (PostgreSQL)               │
│    - 캐시: 당일 스냅샷 (Redis, 선택)                       │
├─────────────────────────────────────────────────────────┤
│ C. 팩터/피처 엔진 (Feature Engine)                        │
│    가격→기술적지표, 재무→퀄리티/밸류, 수급/추정치 가공      │
│    *반드시 point-in-time*: 그 시점에 알 수 있던 값만        │
├─────────────────────────────────────────────────────────┤
│ D. 시그널/스코어링 (Signal & Scoring)                     │
│    레짐 판정 → 섹터 RS → 종목 종합점수 → 게이트 → 랭킹     │
├─────────────────────────────────────────────────────────┤
│ E. 백테스트 엔진 (Backtester)  ★성패의 90%★               │
│    look-ahead/생존편향/비용/유동성 반영, 워크포워드 검증    │
├─────────────────────────────────────────────────────────┤
│ F. 포트폴리오 & 추천 (Portfolio & Recommender)            │
│    내 보유 입력 → 일일 보유/축소/매도/교체 판단            │
├─────────────────────────────────────────────────────────┤
│ G. 프론트엔드 (Dashboard)                                 │
│    레짐 신호등 / 주도섹터 / 매수후보 / 내 포트 진단        │
├─────────────────────────────────────────────────────────┤
│ H. 알림 (Alerting)  — 손절도달/교체신호/레짐변경 푸시      │
└─────────────────────────────────────────────────────────┘
```

> **설계 규칙**: 화면(G)은 *읽기 전용 뷰*여야 합니다. 모든 계산은 D/E/F에서 끝나 있고, 프론트는 결과만 그립니다. 그래야 같은 로직으로 백테스트와 실전이 돌아갑니다(코드 일원화).

## 4. 핵심 엔진 — 7개 레이어 상세

### 4.1 Layer 1 — 매크로 레짐 필터 (위험 ON/OFF 신호등)

**목적**: "지금 주식을 사도 되는 국면인가"를 먼저 판정. risk-off면 시스템 전체가 방어 모드로.

판정에 쓸 입력(예시):
- **추세**: KOSPI 종가 vs 200일 이동평균(MA200) — 위면 우호
- **변동성**: VKOSPI 수준/급등 여부 — 급등 시 risk-off 가중
- **환율**: USD/KRW 추세 — 원화 급격 약세는 외국인 이탈·위험회피 신호
- **글로벌 위험**: 미 하이일드 스프레드(FRED), DXY, 미 10년물 — 스프레드 급확대는 위험
- **수급**: 외국인 전체 순매수 추세 — 지속 매도면 경계

**산출**: 단순 점수제 권장(과적합 방지). 예) 위 지표 각각 +1/0/−1 → 합산.
- 합산 ≥ +2 → **Risk-On** (정상 매수, 총노출 100%)
- −1 ~ +1 → **Neutral** (총노출 축소, 50~70%)
- ≤ −2 → **Risk-Off** (신규매수 중단, 현금/방어, 기존 보유 타이트하게)

> 레짐은 *총노출(gross exposure)*과 *신규진입 허용 여부*를 결정합니다. 종목 선정 로직은 그대로 두고, "엑셀러레이터를 얼마나 밟을지"만 바꾸는 겁니다.

### 4.2 Layer 2 — 섹터/테마 로테이션 (사냥터 좁히기)

**목적**: 돈이 몰리는 주도 섹터로 유니버스를 압축. 추세추종은 *강한 곳에서 더 강한 종목*을 사는 게 정석.

방법:
- KRX 업종지수(또는 테마 그룹)별 **상대강도(RS)** 계산.
  - `RS_score = w1·수익률(3M) + w2·수익률(6M) + w3·(지수 > MA60 여부)`
  - 여기에 **섹터 수급**(외국인+기관 순매수/섹터 시총)을 가산하면 한국형으로 강해집니다.
- 섹터를 RS로 정렬 → **상위 N개(예: 상위 30%)만 "활성 사냥터"**로 지정.
- 하위 섹터의 종목은 아무리 개별 점수가 좋아도 *원칙적으로 신규매수 제외*(역추세 함정 회피).

> 산출물: 대시보드의 "현재 주도 섹터 TOP" 패널. 매수 후보는 항상 이 안에서만 나옵니다.

### 4.3 Layer 3 — 종목 선정 (멀티팩터 종합점수)

주도 섹터 안의 종목들에 대해, **게이트(필수조건)**를 통과한 것만 **종합점수**로 랭킹합니다. (구체 공식은 5·6장)

5대 팩터 그룹과 권장 가중치(시작점, 추후 검증으로 조정):

| 팩터 그룹 | 가중치 | 왜 |
|---|---|---|
| 추세·모멘텀 (Trend & Momentum) | **35%** | 시스템의 엔진. 가고 있는 것에 올라탐 |
| 상대강도 (Relative Strength vs 시장/섹터) | **20%** | 같은 추세라도 더 센 놈 |
| 퀄리티·성장 (Quality & Growth) | **20%** | 지뢰 회피 + 추세 지속력 |
| 수급 (외국인·기관 순매수) | **15%** | 한국형 알파 |
| 밸류에이션 적정성 (Valuation sanity) | **10%** | "비싸도 너무 비싸진 않게" 안전판 |

### 4.4 Layer 4 — 진입 타이밍

종합점수 상위 종목이라도 *지금 가격 자리*가 나쁘면 기다립니다. 추세추종형 진입 패턴(택1 또는 조합):
- **정배열 + 눌림 후 반등**: P > MA20 > MA60 > MA120 이고, MA20 근처로 눌렸다가 반등 캔들.
- **신고가 돌파(브레이크아웃)**: 최근 N일(예 60일) 고가 또는 박스 상단 돌파 + 거래량 동반.
- **필수 동반조건**: 거래량 평균 대비 증가(돌파의 진위), 당일 ±상한가/단일가 이슈 없음.

> "지금 살만한가?"의 최종 게이트가 여기입니다. *좋은 종목 ≠ 지금 사는 종목*. 점수 상위지만 과열(예: 20일선 대비 +X% 이격 과대)이면 "관심"으로만 표시하고 진입 보류.

### 4.5 Layer 5 — 포지션 사이징 & 리스크 (가장 과소평가되는 단계)

수익은 종목 선택이 만들지만, **생존은 사이징이 만듭니다.**

- **변동성 타기팅**: 비중 ∝ 1 / 변동성(ATR 또는 실현변동성). 변동성 큰 종목은 작게, 작은 종목은 크게 → 각 포지션의 *리스크 기여도*를 균등화.
- **고정 분율 리스크(fixed-fractional)**: 한 거래에서 잃을 수 있는 돈을 *총자산의 일정 %로 고정*(예: 0.5~1%). 손절선까지 거리(진입가−손절가)가 정해지면 매수 수량이 역산됩니다.
  - `매수금액 = (총자산 × 리스크%) / (진입가 대비 손절폭%)`
- **한도(Caps)**:
  - 종목당 최대 비중(예: 10~15%)
  - **섹터당 최대 비중(예: 30~40%)** — 한 테마 쏠림 방지(상관 리스크의 핵심)
  - 총노출은 Layer 1 레짐에 연동(Risk-On 100%, Neutral 60%, Risk-Off ↓)

### 4.6 Layer 6 — 청산 규칙 (사전에 정의, 감정 개입 차단)

매도 트리거(하나라도 충족 시 행동):
- **손절**: 진입 시 정한 손절가 도달(예: 진입가 −2×ATR, 또는 −8% 등 고정).
- **추세 추적 손절(Trailing stop)**: 고점 대비 −X%, 또는 종가가 MA20/MA60 하향 이탈.
- **상대강도 악화**: 종목 RS 랭크가 섹터/시장 대비 하위로 추락.
- **수급 이탈**: 외국인·기관이 순매도로 전환되어 지속.
- **펀더멘탈 훼손**: 어닝 쇼크, 추정치 급격 하향, 악재 공시.
- **익절(부분)**: 큰 추세에선 전량매도보다 *분할 익절 + 트레일링*으로 추세를 끝까지(추세추종의 손익비는 여기서 나옴 — 너무 일찍 팔면 엣지가 죽음).

### 4.7 Layer 7 — 포트폴리오 모니터링 (보유/축소/매도/교체 엔진) ← 요청의 핵심 기능

매일(장 마감 후 배치) 보유 종목 각각에 대해 **건강검진 점수**를 다시 계산합니다.

각 보유 종목에 대해:
1. **현재 종합점수 재계산** (Layer 3과 동일 로직).
2. **훼손 신호 체크**: 추세훼손/RS하락/수급이탈/손절도달/펀더 악화 (Layer 6).
3. **의사결정 매트릭스**:

| 상태 | 행동 |
|---|---|
| 훼손 신호 없음 + 점수 여전히 상위 | **HOLD (보유)** |
| 일부 약화(점수 하락, 손절 전) | **TRIM (축소)** — 리스크 절반으로 |
| 손절/추세훼손/펀더 악재 | **SELL (매도)** |
| 매도/축소로 현금 발생 + 더 나은 후보 존재 | **ROTATE (교체)** |

**교체(ROTATE) 판정 — "갈아탈까?"의 규칙화**:
> 신규 후보의 종합점수가 보유 종목 점수를 **임계치(예: +10점) 이상** 초과하고, *동시에* 보유 종목에 약화 신호가 있을 때만 교체.
> — 단순히 "더 좋은 게 보인다"고 갈아타면 회전율·비용만 폭증하고 추세를 놓칩니다. **마찰(임계치)을 둬서 잦은 교체를 억제**하는 게 핵심. (거래비용·세금이 교체의 적입니다.)

대시보드 산출물: 보유 종목별 *신호등(초록 보유 / 노랑 축소 / 빨강 매도)* + 교체 추천 시 "A → B 갈아타기 (점수 72→85, 이유: A 수급이탈·추세훼손)".

## 5. 팩터 정의 (구체 공식 — 구현용)

모든 팩터는 **유니버스 내 횡단면 백분위(percentile rank)** 또는 z-score로 정규화하고, 극단값은 1/99%에서 윈저라이즈(winsorize)합니다. 그래야 서로 다른 단위의 팩터를 합산할 수 있습니다.

**추세·모멘텀**
- `MOM_12_1 = P[t-21] / P[t-252] − 1` (최근 1개월 제외 12개월 수익률 — 단기 반전 노이즈 제거)
- `MOM_6 = P[t] / P[t-126] − 1`
- `MA정배열점수 = (P>MA20) + (MA20>MA60) + (MA60>MA120) + (MA120>MA200), 0~4`
- `52주고가근접도 = P[t] / max(P[t-251 … t])` (1에 가까울수록 강세)
- `ADX(14)` (추세 강도 — 횡보장 필터)

**상대강도(RS)**
- `RS_시장 = 수익률_종목(N) − 수익률_KOSPI(N)`
- `RS_섹터 = 수익률_종목(N) − 수익률_섹터지수(N)`
- (선택) RS 라인의 기울기(우상향 여부)

**퀄리티·성장**
- `ROE`, `ROIC = NOPAT / 투하자본`
- `영업이익률`과 그 *추세*(YoY 개선?)
- `부채비율`, `이자보상배율` (재무건전성)
- `FCF > 0` 여부
- `매출 YoY`, `영업이익 YoY`, `EPS YoY`
- (유료 데이터 붙이면) `EPS 추정치 1M 변화율` ← 강력

**수급**
- `수급강도 = Σ(외국인+기관 순매수, 최근 20일) / 시가총액`
- (선택) 외국인 보유비중의 *증가 추세*

**밸류에이션 적정성**
- `PER`, `PBR`, `EV/EBITDA` — **섹터 내 백분위**로(섹터마다 적정 수준이 다름)
- `PEG = PER / 이익성장률` (성장 대비 비싼지)
- 용도: 점수에 *역방향*으로 소폭 반영. 너무 비싼 종목에 페널티(추세는 좋지만 과열인 것 제어).

## 6. 종합 스코어 모델 — "지금 살만한가"의 최종 산식

2단계입니다: **(A) 게이트(필수조건, 하나라도 실패하면 탈락) → (B) 점수(통과한 것만 랭킹)**.

### (A) 하드 게이트 (Necessary conditions — 전부 통과해야 후보)
1. **레짐**: Risk-Off 아님 (Off면 신규 후보 0).
2. **섹터**: 주도 섹터(RS 상위)에 속함.
3. **유동성**: 20일 평균 거래대금 ≥ 기준(예: 50억 원) — *체결·슬리피지 보장*. (이거 없으면 백테스트는 되는데 실전에서 못 산다.)
4. **추세 정상**: 종가 > MA120(또는 MA200) 그리고 MA 정배열점수 ≥ 3.
5. **지뢰 회피**: 관리종목/투자경고/거래정지/자본잠식 아님, 감사의견 적정, 적자 지속 아님(또는 명확한 흑자전환).
6. **가격 이슈 없음**: 당일 거래정지/단일가/상장 직후 등 이상상태 아님.

### (B) 종합점수 (0~100)
```
Score = 100 × (
    0.35 × pct(추세·모멘텀)
  + 0.20 × pct(상대강도)
  + 0.20 × pct(퀄리티·성장)
  + 0.15 × pct(수급)
  + 0.10 × (1 − pct(밸류_비쌈정도))   ← 비쌀수록 감점
)
```
- `pct(...)` = 유니버스 내 백분위(0~1).
- 게이트 통과 종목을 Score로 정렬 → **상위 K개(예: 15~20개)**가 "지금 살만한 종목 리스트".
- **진입 타이밍 게이트(Layer 4)**까지 통과한 종목에는 **"BUY NOW"**, 점수는 높지만 자리가 안 좋으면 **"WATCH"** 라벨.

> 가중치·임계치·룩백기간은 *반드시 백테스트로 검증/조정*하세요. 단, 자유 파라미터가 많을수록 과적합 위험이 커집니다 — **단순하게 시작**하고, 추가하는 모든 복잡성은 *아웃오브샘플 성과 향상*으로 정당화하세요.

## 7. 백테스팅 — 여기서 성패의 90%가 갈린다

대부분의 아마추어 퀀트가 망하는 지점입니다. 화려한 백테스트의 99%는 아래 함정 중 하나입니다.

1. **생존편향(Survivorship bias)**: 현재 상장 종목만으로 테스트하면, *상장폐지된 종목들*(= 망한 종목)이 빠져서 성과가 과대평가됩니다. → **상폐 종목 포함된 유니버스**를 쓰거나, 최소한 한계를 인지하세요.
2. **미래참조(Look-ahead bias)**: 그 시점에 *알 수 없던 정보*를 사용. 한국 특화 함정 둘:
   - 재무는 **DART 공시일 기준**으로(분기 실적은 발표 전까지 모름).
   - 수급/지수도 *마감 후 확정값*을 *마감 전에* 쓰지 않도록.
3. **거래비용·세금 누락**: 한국은 비용이 작지 않습니다.
   - 매매수수료(증권사별), **매도 시 증권거래세**(코스피/코스닥 약 0.15% 내외 — *매년 변동, 시행 시점 확인*), **슬리피지**(특히 저유동성).
   - 회전율이 높은 전략은 비용으로 알파가 통째로 사라집니다. → 비용을 **보수적으로 크게** 잡고도 살아남는지 보세요.
4. **유동성 비현실**: 일 거래대금 10억짜리 종목을 5억어치 산다고 가정하면 비현실. *체결 가능 물량 한도*를 백테스트에 반영.
5. **과적합(Overfitting)**: 파라미터를 과거에 맞춰 깎으면 미래엔 안 통합니다.
   - **워크포워드(walk-forward)** 검증: 과거 구간에서 정하고, *그 다음 미지의 구간*에서 평가.
   - **아웃오브샘플** 분리, 다중검정 보정(파라미터 수십 개 돌려 best 하나 고르는 건 자기기만).
   - 파라미터 민감도 분석: 값을 약간 바꿔도 성과가 *완만하게* 변해야 robust(절벽처럼 변하면 과적합).

**볼 지표**: CAGR, **MDD(최대낙폭)**, **샤프/소르티노**, 손익비(payoff ratio), 승률(참고용), 회전율, 비용 후 순수익, 구간별/레짐별 성과.

> CEO의 한 줄: **"백테스트에서 못 살아남는 전략에 실제 돈을 넣지 않는다. 그리고 백테스트가 너무 좋으면 의심부터 한다."**

- Layer 7(보유/축소/매도/교체), 매수후보 랭킹, 알림.
- *산출물*: 내 포트 입력 → 매일 액션 리스트.

**Phase 3 — 대시보드(프론트) (2~3주)**
- 레짐 신호등 / 주도섹터 / 매수후보(BUY·WATCH) / 내 포트 진단(신호등·교체추천).

**Phase 4 — 고도화 (지속)**
- 유료 컨센서스(추정치 리비전 팩터) 추가, 팩터 추가/검증, 파라미터 재최적화, 페이퍼 트레이딩으로 라이브 검증.

> **자본 배분 원칙**: 유료 데이터·복잡한 ML은 *Phase 1에서 단순 전략이 먹힌다는 게 확인된 다음*. 순서를 지키세요.

## 10. 기술 스택 제안 (Claude Code 빌드용)

- **백엔드/연산**: Python — pandas, numpy, **pykrx/FinanceDataReader/OpenDartReader/ecos/fredapi**, 백테스트 **vectorbt 또는 backtrader**.
- **스케줄러**: cron(단순) 또는 Prefect/Airflow(확장 시). 매일 장 마감 후(예: 16:00~18:00) 배치.
- **저장소**: PostgreSQL(정형) + Parquet 파일(시계열). 규모 커지면 TimescaleDB.
- **API 서버**: FastAPI (연산 결과를 프론트에 제공).
- **프론트엔드**: React + 차트 라이브러리(예: lightweight-charts/Recharts). 신호등·랭킹·포트 진단 위주의 *읽기 전용 대시보드*.
- **알림**: 텔레그램 봇 / 이메일 / 웹푸시 — 손절도달·교체신호·레짐변경.
- **비밀관리**: API 키는 환경변수/시크릿. *코드·깃에 키를 절대 넣지 말 것.*

> Claude Code에 넘길 때 순서: ① 데이터 어댑터(각 소스별 1파일) → ② 팩터 엔진 → ③ 백테스터 → ④ 스코어/추천 → ⑤ FastAPI → ⑥ 프론트. **모듈 경계를 명확히** 지시하면 디버깅이 쉽습니다.

## 11. 리스크 & 한계 (꼭 읽고 시작하세요)

- **이건 점쟁이가 아니라 의사결정 지원 도구입니다.** "지금 살만한 종목"은 *확률적으로 유리한 후보*이지 보장이 아닙니다. 최종 판단·주문은 사람이.
- **과거 성과는 미래를 보장하지 않습니다.** 시장 구조는 변합니다(레짐 변화, 규제, 유동성). 정기 재검증·재최적화가 필요합니다.
- **추세추종은 횡보장에서 고전합니다.** 추세가 없는 박스권에선 잔손실(휩쏘)이 쌓입니다 — 이건 버그가 아니라 *비용*이며, 큰 추세 한 번으로 회수하는 구조임을 이해하고 견뎌야 합니다.
- **과적합이 가장 큰 적입니다.** 백테스트가 좋을수록 의심하세요. 단순함을 유지하고, 모든 복잡성은 아웃오브샘플 성과로 증명하세요.
- **실거래 전 페이퍼 트레이딩**으로 라이브 데이터·체결·지연을 반드시 검증하세요. 소액으로 시작하세요.
- 저는 투자자문업자가 아니며 이 문서는 일반적 시스템 설계 정보입니다. 실제 투자·세무 판단은 본인 책임이며, 필요하면 전문가(세무사 등) 확인을 권합니다.
